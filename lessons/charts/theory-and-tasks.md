# React Практика: Графики и Визуализация данных

В этом разделе мы научимся создавать графики с нуля, используя CSS, SVG и JS, не прибегая к тяжелым библиотекам типа Recharts или Chart.js. Это отличная тренировка для понимания того, как работают координаты, SVG-пути и масштабирование данных.

---

## 1. Задача 1: Простой Bar Chart (CSS)

Самый базовый тип графика для сравнения величин.

### Требования:
1.  **Входные данные**: Массив объектов `{ label: string, value: number }`. (Например: продажи по дням недели).
2.  **Реализация**:
    *   Используйте `flexbox` для выравнивания столбиков.
    *   Высота каждого столбика должна вычисляться относительно максимального значения в данных (чтобы самый высокий был 100%).
    *   Под каждым столбиком должна быть подпись (`label`).
    *   При наведении на столбик показывать точное значение (`title` или кастомный тултип).
3.  **Стили**: Столбики должны быть анимированы при появлении (расти снизу вверх).

## 2. Задача 2: Line Chart (SVG)

Линейный график для отображения динамики во времени.

### Требования:
1.  **Входные данные**: Массив чисел (например, курс валюты за 10 дней).
2.  **Реализация**:
    *   Используйте SVG `<svg>`.
    *   Вам нужно сгенерировать атрибут `d` для тега `<path>`. Команда `M x y` (move to) и `L x y` (line to).
    *   Ось X: равномерно распределите точки по ширине контейнера.
    *   Ось Y: масштабируйте значения от `min` (низ) до `max` (верх) графика.
3.  **Интерактивность**: Добавьте точки (`<circle>`) на изгибах линии. При клике на точку выводить значение в консоль.

---

## 3. Задача 3: Donut Chart / Pie Chart (SVG)

Круговая диаграмма для отображения долей.

### Требования:
1.  **Входные данные**: Массив `{ color: string, value: number }`.
2.  **Реализация**:
    *   Используйте SVG `<circle>`.
    *   Используйте трюк с `stroke-dasharray` и `stroke-dashoffset`.
    *   Длина окружности = `2 * PI * R`.
    *   Каждый сегмент — это круг с нужным смещением (`offset`).
3.  **Анимация**: Круг должен "заполняться" при загрузке.

## 4. Задача 4: Stacked Bar Chart

Столбчатая диаграмма с накоплением, показывающая состав каждой категории.

### Требования:
1.  **Входные данные**: Массив объектов `{ year: number, mobile: number, desktop: number, tablet: number }`.
2.  **Реализация**:
    *   Каждый столбик состоит из нескольких разноцветных сегментов.
    *   Общая высота столбика должна соответствовать сумме (mobile + desktop + tablet).
    *   Сегменты накладываются друг на друга (или идут друг за другом в flex-column).
3.  **Легенда**: Добавить легенду с цветами категорий под графиком.

## 5. Задача 5: Scatter Plot (Точечная диаграмма)

График для поиска корреляции между двумя переменными (X и Y).

### Требования:
1.  **Входные данные**: Массив точек `{ x: number, y: number, category: 'A' | 'B' }`.
2.  **Реализация**:
    *   SVG или абсолютное позиционирование `div`'ов внутри контейнера.
    *   Нужно нормализовать (масштабировать) координаты: `Math.max` по X и по Y определяют границы графика.
    *   Цвет точки зависит от `category`.
3.  **Сетка**: Нарисовать фоновую сетку (grid lines) с шагом, например, 10% или 50px.

## 6. Задача 6: Area Chart (График с областью)

Похож на Line Chart, но область под линией закрашена.

### Требования:
1.  **Входные данные**: Массив чисел.
2.  **Реализация**:
    *   SVG `<path>`.
    *   В отличие от Line Chart, путь должен быть замкнут: начинаться слева-внизу, идти по точкам, спускаться вправо-вниз и возвращаться в левый-низ (`Z` command).
    *   Использовать `<linearGradient>` в SVG для заливки (сверху насыщенный цвет, снизу прозрачный).
3.  **Сглаживание**: (Advanced) Вместо ломаных линий использовать кривые Безье (Cubic Bezier) для плавности графика.

## 7. Задача 7: Candlestick Chart (Японские свечи)

Сложный финансовый график, отображающий Open, High, Low, Close цены (OHLC).

### Требования:
1.  **Входные данные**: Массив `{ time: string, open: number, high: number, low: number, close: number }`.
2.  **Реализация**:
    *   Каждая "свеча" состоит из:
        *   Тонкой линии (фитиль) от `low` до `high`.
        *   Прямоугольника (тело) от `Math.min(open, close)` до `Math.max(open, close)`.
    *   **Цвет**:
        *   Зеленый, если `close > open` (цена выросла).
        *   Красный, если `close < open` (цена упала).
3.  **Скролл**: Реализовать горизонтальный скролл контейнера, если свечей много.
