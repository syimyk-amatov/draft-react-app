# React Практика: Component Communication, Events, State

## 1. Теория

### State (Состояние)
Состояние — это данные, которые управляют поведением и отображением компонента. Когда состояние меняется, компонент рендерится заново.
В функциональных компонентах мы используем хук `useState`.

```tsx
const [count, setCount] = useState(0);
// count - текущее значение
// setCount - функция для обновления значения
```

### Events (События)
В React события именуются в camelCase (например, `onClick`, `onChange`).
Мы передаем функцию-обработчик в атрибут события.

```tsx
<button onClick={() => setCount(count + 1)}>Нажми меня</button>
```

### Component Communication (Взаимодействие компонентов)

**1. Родитель -> Ребенок (Props)**
Данные передаются от родителя к ребенку через пропсы (свойства).

```tsx
// Родитель
<ChildComponent message="Привет" />

// Ребенок
const ChildComponent = ({ message }) => {
  return <div>{message}</div>
}
```

**2. Ребенок -> Родитель (Callbacks)**
Чтобы передать данные наверх, родитель передает ребенку функцию через пропсы. Ребенок вызывает эту функцию, передавая данные аргументом.

```tsx
// Родитель
const handleUpdate = (newValue) => {
  console.log("Получено от ребенка:", newValue);
}
return <ChildComponent onUpdate={handleUpdate} />

// Ребенок
const ChildComponent = ({ onUpdate }) => {
  return <button onClick={() => onUpdate("Данные")}>Отправить</button>
}
```

## 2. Практическая задача: "Голосование за фрукт"

Вам нужно реализовать простое приложение для голосования.

### Требования:

1.  **Компонент `VotingApp` (Родитель)**
    *   Хранит состояние (state): список фруктов и количество голосов за каждый.
    *   Пример структуры стейта:
        ```js
        [
          { name: "Яблоко", votes: 0 },
          { name: "Банан", votes: 0 },
          { name: "Апельсин", votes: 0 }
        ]
        ```
    *   Отображает заголовок "Голосование".

2.  **Компонент `FruitCard` (Ребенок)**
    *   Принимает через props:
        *   `name` (название фрукта)
        *   `votes` (количество голосов)
        *   `onVote` (функция-колбэк для голосования)
    *   Отображает название и текущее количество голосов.
    *   Имеет кнопку "Голосовать", которая вызывает `onVote`.

3.  **Компонент `WinnerDisplay` (Ребенок)**
    *   Принимает список фруктов.
    *   Вычисляет и отображает фрукт-победитель (тот, у которого больше всего голосов). Если голосов 0, пишет "Пока нет голосов".

### Шаги выполнения:
1. Создайте папку `src/lessons/voting`.
2. Создайте файлы компонентов.
3. Реализуйте логику передачи данных и обновления стейта.
